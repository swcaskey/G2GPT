What is HTML
Hypertext Markup Language (HTML) is the standard language for creating and structuring content on the web.
Defines the structure and layout of web pages.
Organizes text, images, links, and other multimedia elements.
Works alongside CSS (for styling) and JavaScript (for interactivity).

Characteristics:
 High-Level Language:   Abstracts complex hardware details.
 Markup Language:        Uses tags to annotate text and define elements.
 Multi-Paradigm:            Supports both object-oriented and functional programming styles.

Key Concepts in JavaScript Testing (Code Testing in general)
Testing Basics
Purpose:  Validate program correctness with     minimal, effective test cases.
Challenges:
Testing cannot cover all scenarios; focus on diverse, meaningful inputs.
Boundary cases and code coverage are critical for identifying defects.
Key Goals:
Boundary Case Coverage: Test edge cases where input/output behavior is complex.
Code Coverage: Ensure all instructions and branches are executed at least once.
2. Key Testing Considerations
Input/Output Mapping:
Abstract programs as mappings from inputs ("domain") to outputs ("codomain").
Context-Dependent Validity:
Example: Valid dates depend on the month and year (e.g., February 29 in leap years).

--- Slide 23 ---
Sample Test Case - Validating CSV File Input
1.1.3 JavaScript Testing
23
Objective:    Ensure that the user provides a file with a .csv extension.
Function to Test:
Test Cases:

Explanation:
describe('CSV Validator', ...) -> Groups all tests related to the CSV validation functionality.
First it Block:
Purpose: Test that isValidCSV returns true when a valid .csv file path is provided.
Expectation: result should be true.
Second it Block:
Purpose: Test that isValidCSV returns false when a non-.csv file path is provided.
Expectation: result should be false.
  function isValidCSV(filePath) {
    return filePath.endsWith('.csv');
  }
  describe('CSV Validator', () => {
    it('should return true for a valid CSV file', () => {
      const filePath = 'data/sample.csv';
      const result = isValidCSV(filePath);
      expect(result).toBe(true);
    });
  
    it('should return false for an invalid CSV file', () => {
      const filePath = 'data/sample.txt';
      const result = isValidCSV(filePath);
      expect(result).toBe(false);
    });
  });

Key Features of Node.js
1.2 NodeJS
28
Event-Driven, Non-Blocking I/O
Single-Threaded Execution:
Node.js runs on a single thread for program control.
Long-running tasks like I/O or network calls are handled asynchronously.
Asynchronous Programming:
Code executes continuously without being blocked by tasks like file reading or API calls.
Simplifies development compared to traditional multi-threading.
Example: Console Input with Readline Module
Readline Module:
Allows reading input directly from the console, one line at a time.
Code Example:
const readline = require('readline');
const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});
rl.question('What is your name? ', (answer) => {

  rl.close();
});
