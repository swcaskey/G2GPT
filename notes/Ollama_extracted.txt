What is Ollama
• Ollama = local LLM runtime + model manager
• Lets you download (“pull”), run, and serve LLMs on your machine
• Provides:
• CLI for interactive use
• Local HTTP API for programmatic calls (your backend can call it)
• http://127.0.0.1:11434 by default

--- Page 5 ---
Why Ollama
• There are multiple tools for running LLMs locally:
• vLLM
• High-performance GPU inference
• Optimized for high concurrency and throughput
• Commonly used when deploying LLMs as a scalable service
• llama.cpp
• Lightweight C/C++ inference engine
• Efficient on CPU and low-resource machines

--- Page 6 ---
Why Ollama
• Ollama
• Simple installation across macOS, Windows, and Linux
• Unified interface
• Both CLI and Local HTTP API
• Easy model management (pull, run, list)
• Ollama is the best match of our course goal: Focus on system design,
not deployment complexity or inference optimization

Model Selection
• Smaller models
• Faster, lower memory usage
• Good for basic chat, simple reasoning, quick demos
• Larger models
• Better reasoning and generation quality
• Slower and more memory-intensive

--- Page 10 ---
Model Selection
Total memory ≈ Model weights + KV cache + Runtime overhead
≤ 8 GB RAM
→ Use small models (e.g., ~2B–4B parameters)
16 GB RAM
→ Mid-size models (~7B parameters) usually work well
32 GB+ RAM or GPU available
→ Larger models are possible, but expect higher latency

Customize Your Model
• Ollama provide Modelfile, letting you define default behavior for a
model.
• Example:
### Modelfile
FROM deepseek-r1:14b
# sets the temperature to 0.2 [higher is more creative, lower is
more coherent]
PARAMETER temperature 0.2
# sets the context window size to 4096, this controls how many
tokens the LLM can use as context to generate the next token
PARAMETER num_ctx 4096

--- Page 15 ---
Customize Your Model
Trade-offs:
• Context length
• Longer context allows the model to take in more information
• Comes with higher memory usage due to a larger KV cache
• Can significantly increase latency, even for short inputs
• Very long contexts may also lead to context rot, where earlier information becomes
less effective
• Model size
• Larger models may provide better reasoning and generation quality (maybe)
• Require more RAM / VRAM to load model weights
• Smaller models are often fast, stable, and sufficient for many simple tasks

--- Page 16 ---
Customize Your Model
Trade-offs:
• Practical takeaway
• Context length and model size are not “the bigger, the better”
• They should be chosen based on task requirements and hardware
constraints
• Start from a 1~4B model with a context length of 4096, then adjust
as needed

--- Page 17 ---
Possible Issues
• RAM/VRAM issue
• Model may be too large for your machine
• Symptoms: very slow response, model fails to load, or process gets killed
• Solution: choose a smaller model or close other memory-intensive
applications
• Port unavailable
• Default port is 11434
• Error occurs if the port is already in use
• Solution: stop the existing Ollama service or check which process is using
the port
