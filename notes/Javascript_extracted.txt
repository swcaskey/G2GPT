Function Declaration: (1) Explicit Declaration
Explicit declaration by using the function keyword, followed by a unique function name, a list of parameters (that might be empty), and a block of statements surrounded by curly braces
Example:
1	function summation(a, b, c) {
2	  let s = a + b + c;
3	  return s;
4	}		🡨 Note: no semicolon after the ending curly brace of a function
5	let sum = summation(2,3,4);


--- Slide 18 ---
Function Declaration: (2) Anonymous Declaration
Function defined without a name (nameless);may be assigned to a variable or passed in as an argument to another function
Example:
1	let summation = function(a, b, c) {
2	  let s = a + b + c;
3	  return s;
4	};		🡨 Ends with a semicolon because it is an executable statement
5	let sum = summation(2,3,4);
			   🡬 Called using the variable name to which it was assigned


--- Slide 19 ---
Function Declaration: (3) Arrow Declaration - 1
Arrow functions so called because they use a token => are also a kind of anonymous functions
When using arrow functions, it is not necessary to type in the keywords function and return, or use curly braces
Example:
	  let summation = (a, b, c) => { return a + b + c };
Curly braces around the function body are not required when the function has a single statement, so instead we can write:
	  let summation = (a, b, c) => a + b + c;
Parentheses around the parameters are not required when the function has a single parameter:
	  let cubed = x => x*x*x;
Parentheses are required when the function has no parameters:
	  let greeting = () => window.alert("Hello World!");

The arguments Object & The Rest Operator
The arguments object is an array-like object that contains the values of the arguments passed to the function
Includes excess arguments that were not listed when the function was declared
	function summation(a, b, c) {
	  // instead of simply returning a + b + c, let's try:
	  return arguments[0] + arguments[1] + arguments[2];
	}
	console.log(summation(1, 2, 3));		// output: 6
The arguments object is considered outdated and the new way is to use the rest operator:
	function summation(...allArgs) {
	  return allArgs.reduce((previous, current) => {
	    return previous + current;
	  });
	}
	 console.log(summation(1, 2, 3));	// output: 6

Errors and Exceptions
An exception is any unusual event, erroneous or not, that is detectable by either hardware or software and that may require special processing
 Exception examples:
Number ranges are exceeded in arithmetic operations
Division by zero is attempted
An array subscript reached beyond the range
A possible case is missing in a multiple selector construct (the switch statement)
The user performed an unexpected action
Other hardware errors or user-defined conditions …
Exception handling is the special processing that may be required when an exception is detected
This processing is done by a code unit or segment called an exception handler 
Different types of exceptions require different exception handlers

Property Accessors
A named data property associates a key with a value and you use the key to manipulate the value directly

 Example:

      this._score = 0;
   }
   get mScore() { return this._score; }	 // getter method
   set mScore(increment) {			 // setter method
      this._score += increment;
   }
}
Now we can increment the score by saying:
	member2.mScore = 3;


get and set are special keywords in JS. While they look like functions, they do not follow the typical function syntax because they allow you to bind the property access and assignment behavior to custom logic, without calling them as regular functions.
