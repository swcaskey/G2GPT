# ====================
# Running the Application
# ====================
1. Open a terminal in the project root directory
2. Start the server:
	node server.js
3. Open your browser and navigate to:
   http://localhost:3000




# ====================
# Application Pages
# ====================
Public Area:
- Landing Page:        http://localhost:3000/
- Sign Up:             http://localhost:3000/signup
- Log In:              http://localhost:3000/login

Authenticated Area:
- Dashboard:           http://localhost:3000/dashboard
- Log Out:             http://localhost:3000/logout

API Endpoints:
- Health Check:        http://localhost:3000/api/health
- Login History:       http://localhost:3000/logins
- LLM Models:          http://localhost:3000/api/models
- LLM Chat:            http://localhost:3000/api/chat
- User Conversations:  http://localhost:3000/api/conversations
- Conversation Messages: http://localhost:3000/api/conversations/:id/messages




# ====================
# Running Tests
# ====================
Complete Test Suite (51 tests):
  npm test                              # Runs all Jasmine unit tests

Individual Test Files:
  npx jasmine spec/loginSpec.js         # Auth/API unit tests (23 specs)
  npx jasmine spec/dashboardSpec.js     # Dashboard + multi-model unit tests (28 specs)

Cucumber.js (Acceptance Tests):
  npm run test:cucumber                 # All features (10 scenarios total)
  npx cucumber-js features/authentication.feature  # Iteration 1 (3 scenarios)
  npx cucumber-js features/chat.feature            # Iteration 2 (4 scenarios)
  npx cucumber-js features/multi-model.feature     # Iteration Individual (3 scenarios)

Puppeteer (Browser Automation):
  # Puppeteer is integrated with Cucumber.js
  # Tests run automatically with visual demonstrations
  npm run test:cucumber                 # Runs Puppeteer + Cucumber
  npm run test:all                      # Complete test suite (Jasmine + Cucumber/Puppeteer)

Test Breakdown by Iteration:
  Iteration 1 (Authentication):
    - Jasmine: covered in spec/loginSpec.js
    - Cucumber: 3 scenarios in features/authentication.feature
      1. Landing Page Access
      2. Account Registration  
      3. Local Log In and Log Out
  
  Iteration 2 (LLM Chat):
    - Jasmine: covered in spec/dashboardSpec.js
    - Cucumber: 4 scenarios in features/chat.feature
      1. Send a prompt and receive a response
      2. Conversation history is saved
      3. Search conversation history
      4. Resume a conversation from history

  Iteration Individual (Multi-LLM):
    - Select one or two models from dashboard model selector
    - One prompt sends parallel requests to all selected models
    - Responses are labeled by model and saved with model_name in messages
    - Cucumber: 3 scenarios in features/multi-model.feature
      1. Select one or two models from selector
      2. Send one prompt to two models and verify both labels
      3. Reopen conversation and verify persisted model labels
